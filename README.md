# Narrative-Driven Scene Reconstruction

## Project Overview
This project builds an AI system that generates images from narrative text while strictly preserving spatial structures and object mappings. Traditional text-to-image models (like default Stable Diffusion pipelines) often ignore structural relationships, hallucinate objects, or misplace objects entirely. 

This pipeline resolves those flaws by parsing text functionally beforehand:
**Text → Structured Scene Graph → Spatial Layout → Controlled Image Generation**

By forcing an intermediate mapping stage (Scene Graph), the generative diffusion process aligns flawlessly with controllable layout grids resulting in accurate narrative renderings.

### Architecture Pipeline
The system operates as a 6-stage pipeline:
1. **Text Preprocessing:** Tokenization, NLP cleaning.
2. **Semantic Extraction:** Capturing objects, attributes, and precise relationships.
3. **Scene Graph Validation:** Rules-based validation ensuring scene accuracy.
4. **Spatial Layout Generation:** 2D bounding-box spatial engine coordinate math. 
5. **ControlNet Conditioning:** Visual representations injected as generative priors.
6. **Diffusion Generation:** The final rendered semantic image.

## Phase 1 Implementation: NLP Engineering Pipeline
*Assigned to: Person 1 (NLP Engineer)*

This repository currently implements **Stage 1 & 2** of the above pipeline, processing raw image text from the `Visual Genome` dataset into structured feature rules used for spatial parsing.

### What Has Been Built

1. **Dataset Download (`download_dataset.py`)**
   - Automatically downloads the full Visual Genome `region_descriptions.json.zip` dataset.
   - Outputs the extracted dataset to use in actual NLP processing.

2. **Core Extraction Engine (`process_visual_genome.py`)** 
   - **Text Cleanser**: Mapped regex sanitization cleaning out stray punctuation, invalid characters, and malformed spacing. Generates UUID references and writes the unified output to `cleaned_text.txt`.
   - **Part-of-Speech & Dependency Processing**: Utilizes the `spaCy` NLP library (`en_core_web_sm`) to tag linguistic hierarchies.
   - **Object Recognition**: Employs noun chunks for objects while actively filtering out abstract concepts (like "view", "time", "side") ensuring only concrete physical entities are mapped.
   - **Attribute Recognition**: Extracts qualifying adjectives specifically tied to root physical objects.
   - **Spatial Relationship Math**: Evaluates adjacent physical objects sequentially over the text. Uses strict word boundaries matched against prepositions (e.g., "next to", "in front of", "under") to map relationships tightly without greedy false-positives.
   - Generates the final compiled representation to `linguistic_features.json`.

### How to Run Locally

Install the required environment dependencies:
```bash
pip install -r requirements.txt
python -m spacy download en_core_web_sm
```

Initialize the actual dataset:
```bash
python download_dataset.py
```

Run the NLP Extractor:
```bash
python process_visual_genome.py
```

### Outputs Generated
- `dataset/region_descriptions.json`: The full Visual Genome region descriptions dataset.
- `cleaned_text.txt`: The isolated sentence phrases cleaned and mapped by their unique sentence ID.
- `linguistic_features.json`: The fully realized structured output array defining distinct objects, attributes, and inter-object relationships formatted for the layout engine.

## Phase 2 Implementation: Semantic Scene Graph Construction
*Assigned to: Person 2 (Semantic Parsing Engineer)*

This stage converts the structured linguistic features generated in Phase 1 into validated semantic scene graphs that can be consumed by the spatial layout engine in the next stage.

While Phase 1 extracts objects, attributes, and relationships from text, this phase normalizes and validates that information to produce a structured semantic representation of the scene.

The goal of this module is to ensure that the pipeline receives clean spatial information, preventing duplicate objects, abstract entities, or invalid relationships that could break spatial layout generation.

### What Has Been Built

**Semantic Scene Graph Parser (`semantic_module/semantic_parser.py`)**
- Reads the structured output generated by the NLP extraction stage (`linguistic_features.json`).
- Converts extracted linguistic features into structured scene graphs.
- Assigns unique IDs to objects within each scene.
- Ensures that relationships reference valid objects before generating the scene graph.

**Object Normalization (`semantic_module/normalizer.py`)**
- Standardizes object names so that different synonyms refer to the same entity.
- Example transformations include:
  - `man`, `woman`, `guy`, `suspect` → `person`
  - `vehicle`, `automobile` → `car`
  - `bike`, `cycle` → `bicycle`
  - `pistol`, `revolver` → `gun`
- This prevents duplicate semantic objects during layout generation.

**Abstract Object Filtering**
- Removes objects that do not represent physical entities and cannot be placed in a spatial layout.
- Examples removed include terms such as `colour`, `shade`, `edge`, `area`, `portion`, `background`, `scene`, and `image`.

**Spatial Relation Normalization**
- Converts extracted relations into a canonical spatial format required for layout generation.
- Supported spatial relations include: `near`, `left_of`, `right_of`, `above`, `below`, `on`, `under`, `inside`
- Example relation mappings:
  - `next_to`, `beside`, `close_to` → `near`
  - `in` → `inside`
  - `over` → `above`
  - `underneath` → `under`
- Non-spatial relations such as `with`, `has`, `of`, or `wearing` are removed.

**Duplicate Object Removal**
- Ensures each object appears only once in a scene.
- Prevents invalid relations like `person near person` that may arise from synonym normalization.

**Scene Complexity Control**
- Limits each scene to a maximum of 6 objects to ensure stable spatial layout generation in later stages.

**Scene Graph Validation (`semantic_module/validator.py`)**
- Validates each scene graph before saving.
- Ensures:
  - At least one object exists
  - Object names are unique
  - Relationships reference valid objects
  - No self-relations occur
  - Relations belong to the allowed spatial relation set

### How to Run Locally

After running the NLP extraction stage, execute the semantic parser:
```bash
python semantic_module/semantic_parser.py
```

### Outputs Generated

- `scene_graph.json`: A validated semantic scene graph representation of each narrative description.

Example scene graph entry:
```json
{
  "scene_id": 42,
  "objects": [
    {"id": 1, "name": "person"},
    {"id": 2, "name": "car"},
    {"id": 3, "name": "gun"}
  ],
  "relationships": [
    {"subject_id": 1, "relation": "near", "object_id": 2}
  ]
}
```

### Processing Statistics

- **52,299** scene graphs generated
- **495** scenes skipped due to validation rules (invalid relations, duplicate objects, or abstract entities)

The generated `scene_graph.json` file serves as the input for the spatial layout generation module, which converts semantic relationships into 2D spatial coordinates used by the generative model.
